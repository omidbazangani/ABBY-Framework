Trace processing¶

Trace processing tools.

abby.processing.crop_cycles(trace, threshold, samples_per_cycle=1)[source]¶

Crop cycles at beginning and end of trace.

Abby firmware injects 500 NOP instructions before and after cipher execution for alignment and debug purposes. We search for a group of 500*samples_per_cycle samples under the threshold.

Parameters

  • trace ([float] or np.ndarray) – trace to process

  • threshold (float) – threshold for cropping

  • samples_per_cycle (int, optional) – number of samples for each cycle, default to 1

Returns

cropped trace

Return type

np.ndarray

abby.processing.find_clock_freq_phase(trace, freq_estimated=8000000.0, freq_precision=1000.0, threshold=0.0002, sample_rate=250000000.0)[source]¶

Find the CPU clock frequency and phase from the trace.

Compute the fast Fourier transform of the trace then search for the spike corresponding to clock frequency. Using the frequency and phase from this spike, you can align trace with clock cycles.

Using this function to cut clock cycles is rather imprecise when clock jitter occurs. Using the crystal oscillator on a STM32F0 Discovery board, we observed different clock frequency between executing NOP instructions and STR instructions. If you are profiling a small amount of instructions surrounded by NOP instructions, you might want to use abby.processing.crop_cycles() with samples_per_cycle = sample_rate / freq_estimated.

You can visualize what this function does using abby.plot.plot_fourier().

Parameters

  • trace ([float] or np.ndarray) – side-channel trace to process

  • freq_estimated (float, optional) – estimated clock frequency in Hz, defaults to 8 MHz

  • freq_precision (float, optional) – clock frequency precision in Hz, defaults to 1 kHz

  • threshold (float, optional) – threshold line for spikes detection, defaults to 0.0002

  • sample_rate (float, optional) – trace sampling rate in Hz, defaults to 250 MHz

Returns

found frequency and phase

Return type

(float, float)
