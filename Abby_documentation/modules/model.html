Models¶

Models fitting and prediction code.

Abby provides abstraction over common model types to help training them.

class abby.model.CatBoostModel(path, create=False, **kwargs)[source]¶

Bases: abby.model.base.Model

Boosted tree model using CatBoost

By default learning and prediction is done using GPU. To use CPU you can add task_type="CPU" when instancing this class.

The default hyperparameters were optimized with a grid search on the power consumption of an ST STM32F0308 Discovery board.

class abby.model.ELMOModel(create=False)[source]¶

Bases: abby.model.base.Model

Use ELMO model to predict power.

ELMO is a leakage simulator designed for ARM Cortex-M0. See <https://eprint.iacr.org/2016/517/20160529:210156>.

fit(data_files, test_size=0.2, batch_size=100000)[source]¶

Fit model on target using provided data

For profiler support, if you get CUPTI_ERROR_INSUFFICIENT_PRIVILEGES then you might need to add an option to the nvidia kernel module. Edit /etc/modprobe.d/nvidia-kernel-common.conf and add options nvidia "NVreg_RestrictProfilingToAdminUsers=0". Then rebuild your initramfs (update-initramfs -u) and reboot.

  • data_files ([str]) – dataset files

  • test_size (float, optional) – portion of the dataset using for evaluation, default to 0.2

  • batch_size (int, optional) – number of records to combine in a single batch, default to 100000

class abby.model.XgbModel(path, create=False)[source]¶

Bases: abby.model.base.Model

Boosted tree model using XGBoost

By default it will try to use GPU for learning and prediction.

XGBoost is currently working on adding categorical feature support but it is still experimental: <https://github.com/dmlc/xgboost/pull/5949>. For now we recommend using CatBoost.
